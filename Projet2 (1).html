Web Full Stack ; Développement de sites pour PME

Vue d'ensemble du projet

Dans le cadre de ce projet, j'ai travaillé sur la sécurisation d'un site web fictif afin de le protéger contre les menaces courantes telles que les injections SQL, les attaques XSS et les attaques par force brute. L'objectif était de renforcer la sécurité du site tout en maintenant une expérience utilisateur fluide.

Défis rencontrés

  • Injection SQL : Prévenir les attaques permettant l'injection de requêtes malveillantes dans la base de données.
  • Attaques XSS (Cross-Site Scripting) : Protéger le site contre l'injection de scripts malveillants dans les pages web vues par d'autres utilisateurs.
  • Protection contre les attaques par force brute : Limiter les tentatives de connexion pour empêcher les attaques par force brute sur les pages de connexion.

Solution proposée

Pour résoudre ces défis, j'ai mis en place plusieurs solutions :

  • Prévention des injections SQL : Utilisation de requêtes préparées et d'une validation stricte des entrées utilisateur pour éviter les attaques par injection.
  • Protection contre XSS : Validation et assainissement des entrées utilisateur, ainsi que l'utilisation de Content Security Policy (CSP) pour limiter l'exécution de scripts malveillants.
  • Protection contre les attaques par force brute : Implémentation de CAPTCHA sur les pages de connexion et limitation du nombre de tentatives de connexion par adresse IP.

Technologies utilisées

Pour ce projet, j'ai utilisé les technologies suivantes :

  • HTML5, CSS3 : Pour la structure et le design du site web.
  • JavaScript : Pour les validations côté client et la gestion des interactions avec l'utilisateur.
  • PHP/MySQL : Pour le backend et la gestion de la base de données.
  • OWASP ZAP : Outil pour tester la sécurité du site web et identifier les vulnérabilités.
